# Internationalization (i18n)

## Summary

The i18n system translates **UI strings** — admin interface, module chrome, theme frontend. Content translation (pages, categories, tags, etc.) is a separate system documented under [multi-language](multi-language.md).

## Layers

| Layer | Location | Function |
|-------|----------|----------|
| Core admin | `internal/i18n/locales/{lang}/messages.json` | Admin UI and shared strings. `T` helper. |
| Module | `modules/<name>/locales/{lang}/messages.json` | Module-scoped strings. Auto-loaded at init via embedded filesystem. `T` helper. |
| Theme | `themes/<name>/locales/{lang}/messages.json` | Frontend strings with **theme → global** fallback. `TTheme` helper. |

Supported languages out of the box: **English (`en`)** and **Russian (`ru`)**.

## Translation file format

```json
{
    "$schema": "../../../../.schema/i18n-schema.json",
    "language": "en",
    "messages": [
        {"id": "mymodule.title", "message": "My Module", "translation": "My Module"}
    ]
}
```

Required: `language`, `messages`, `messages[].id`, `messages[].translation`. Optional: `$schema`, `messages[].message` (reference text for translators).

## ID naming

Dot-notation: `{namespace}.{category}.{key}`. Common patterns:

- `module.title`, `module.description`
- `module.field_name`, `module.field_name_help`
- `module.error_*`, `module.success_*`
- `module.button_*` or `module.*_button`

## Template helpers

| Helper | Use case | Fallback |
|--------|----------|----------|
| `TTheme` | Theme templates (frontend) | Theme → Global |
| `T` | Admin templates, module chrome | Global only |

```html
<!-- Theme template -->
<a href="{{.URL}}">{{TTheme .LangCode "frontend.read_more"}}</a>

<!-- Admin template -->
<h1>{{T .AdminLang "mymodule.title"}}</h1>
```

## Module integration

```go
import "embed"

//go:embed locales
var localesFS embed.FS

func (m *Module) TranslationsFS() embed.FS {
    return localesFS
}
```

The module registry reads each language directory on init and merges into the translation store.

## Go code usage

```go
import "github.com/olegiv/ocms-go/internal/i18n"

message := i18n.T(lang, "mymodule.success_save")
```

Docs path [sources/docs-i18n.md](../sources/docs-i18n.md) uses the shorter module path `ocms-go/internal/i18n` — both resolve to the same package.

## JSON Schema validation

Schema lives at `.schema/i18n-schema.json`. VS Code binding:

```json
{
    "json.schemas": [
        {"fileMatch": ["**/locales/*/messages.json"], "url": "./.schema/i18n-schema.json"}
    ]
}
```

Validates required fields, ISO 639-1 language codes, and message object structure.

## Adding a new language

1. Create `internal/i18n/locales/{lang}/messages.json` and per-module/theme locale files.
2. Copy English as a template; update `language` field and translate entries.
3. Register the language code in `SupportedLanguages` (`internal/i18n/i18n.go`) and/or via admin UI.
4. Rebuild.
